Interpolations

When you want to include in your template the value of a variable from the context, you refer to the variable in an interpolation. Interpolations are simply the variable's name surrounded by { } characters.

Assuming that our current evaluation context contains a variable language with the value "Omelet", we could write the following.

Omelet template:
I am writing in {language}.
Rendered HTML:
I am writing in Omelet.

As shown, variable values are simply dropped in to fill the placeholders in the template. If you have arrays or complex objects in your context, you can refer to individual elements or properties as you would expect. Arrays are zero-indexed, just like in JavaScript.

Assuming we have in the context an array called people with the value ["Alice","Bob","Carol"], and an object called dog with the value {name: "Fido", age: 3}, we can include these values as shown:

Omelet template:
The second person is {people[1]}.
The dog called {dog.name} is {dog.age} years old.
Rendered HTML:
The second person is Bob.
The dog called Fido is 3 years old.

Interpolations are also how you call macros defined elsewhere in your templates. In this case, interpolations take the form {macroName(arg1,arg2,...)}. If the macro takes no parameters, you must still include the empty parentheses: {macroName()}. Here are some examples.

Omelet template:
>def displayInfo(name,age):
    {name} is {age} years old.

{displayInfo("Reid",22)}
{displayInfo("Luke",8)}
Rendered HTML:
Reid is 22 years old.
Luke is 8 years old.
Omelet is a template language developed by Reid Mitchell.
