Tags

Tags in Omelet render into HTML tags. Tag names always start with a @ character. There are three types of tags in Omelet. They all look similar, but are used in slightly different ways.

The first is the block tag. These tag names are written on their own line, and everything that goes inside the tag is written as an indented block beneath the tag name.

Omelet template:
@div
    This is a block tag. They can span multiple
    lines. Everything inside this indented block
    is part of the the same tag.
@div
    Block tags can be nested, like this.
    @div
        This is a nested block tag.
Rendered HTML:
<div>
    This is a block tag. They can span multiple
    lines. Everything inside this indented block
    is part of the the same tag.
</div>
<div>
    Block tags can be nested, like this.
    <div>
        This is a nested block tag.
    </div>
</div>

The second type of tag is the line tag. For these, the tag name is written at the start of a line, and everything after the tag name ends up on the inside of the tag. These are helpful when the contents of a tag will only span one line, so you don't need to have a whole indented block to contain them.

A single space after a tag name is necessary to separate the tag name from the content of the tag. Any more spaces after the first will be captured as part of the contents of the tag.

Omelet template:
@h1 This is a headline
@b This is some bold text
@ul
    @li This is a list item, as a line tag, within a ul block tag.
Rendered HTML:
<h1>This is a headline</h1>
<b>This is some bold text</b>
<ul>
    <li>This is a list item, as a line tag, within a ul block tag.</li>
</ul>

The third type of tag is the inline tag. These can be written in the middle of a block of text, surrounded by { } characters. These are useful when you want to include a tag around some text without breaking up the text into multiple lines. They are often used with tags like @a, @b, and @span.

Omelet template:
This is some text with {@b bold words} in it.
This text has a {@[href="example.com"] link} in it.
Rendered HTML:
This is some text with <b>bold words</b> in it.
This text has a <a href="example.com">link</a> in it.

In all three types of tag, Omelet supports a special shorthand syntax for combining tags, called a multitag. Multitags are useful when you want to surround a piece of text with several tags at once.

To write a multitag, you combine tag names back to back, separated by a | character. The leftmost tag name will render into the outermost HTML tag, as shown below.

Omelet template:
@b|i|u This text is bold, italicized, and underlined.
@ul
    @li|a[href="example1.com"] This is a list item and a link.
    @li|a[href="example2.com"] This is a list item and a link.
Rendered HTML:
<b><i><u>This text is bold, italicized, and underlined.</b></i></u>
<ul>
    <li><a href="example1.com">This is a list item and a link.</a></li>
    <li><a href="example2.com">This is a list item and a link.</a></li>
</ul>
