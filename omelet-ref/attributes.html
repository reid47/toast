Attributes

HTML attributes can be included on tags immediately following the tag name, between [ ] characters. Within the [ ] characters, attributes look the same as they do in HTML, separated by spaces. Note that Omelet does support boolean attributes, like checked below.

Omelet template:
@a[href="example.com"] This is a link
@h1[class="big"] This is a headline.
@button[id="cool" onclick="doSomething();"] Click me.
@input[type="checkbox" checked]
Rendered HTML:
<a href="example.com">This is a link.</a>
<h1 class="big">This is a headline.</h1>
<button id="cool" onclick="doSomething();">Click me.</button>
<input type="checkbox" checked="checked"/>

Since class and id attributes are used so often, Omelet supports a special shorthand syntax for these, similar to Jade and Haml. class attributes can be written as .className and id attributes can be written as #idName, immediately following the tag name and preceding the [ ] characters for the other attributes. You can include multiple .classNames and #idNames by placing them back-to-back.

Omelet template:
@h1.big This is a headline.
@button#cool[onclick="doSomething();"] Click me.
@a#anchor.class1.class2[href="example.com"] A classy link
Rendered HTML:
<h1 class="big">This is a headline.</h1>
<button id="cool" onclick="doSomething();">Click me.</button>
<a id="anchor" class="class1 class2" href="example.com">A classy link</a>
