Raw Tags

Sometimes, you'll want to include text in a template that you don't want parsed by the Omelet parser. For example, you may want to include inline CSS in a style tag or inline JavaScript in a script tag. Without raw tags, the use of the { character in each of these languages would confuse the Omelet parser.

In these cases, you can use raw tags. Raw tags can have names and attributes the same way normal tags do. However, after the tag names and attributes, the character sequence ``` marks the beginning of the raw text. Everything up until the next ``` sequence will be parsed as a raw string and passed verbatim to the rendered HTML.

Although indentation is mostly ignored within raw tags, Omelet does require that the closing ``` sequence is at the same level of indentation as the opening @ character for the raw tag.

In fact, all of the Omelet examples in these language reference pages are written using raw tags. Without raw tags, there would be no way to output actual Omelet code in templates. However, at the moment, there is still no way to include the character sequence ``` within a raw tag, so the example below uses ``. Just know that when actually writing Omelet, you must use ```.

Omelet template:
@script[type="text/javascript"] ``
    function hello(x) {
        console.log("hello, "+x);
    }
``
Rendered HTML:
<script type="text/javascript">
    function hello(x) {
        console.log("hello, "+x);
    }
</script>
